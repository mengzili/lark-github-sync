Lark–GitHub Sync

Set up in ~5 minutes. Everything happens in the browser — nothing to install.

1
Use this template ~10 sec

Create a private copy in your GitHub org. The name, description, and visibility are pre-filled — just pick your org and click Create repository.

Use this template →

Once created, paste your new repo URL here — the links below will update automatically:

2
Create a Lark app ~3 min

Follow the sub-steps below. Navigation paths show exactly where to click.

Open Feishu Console → Open Lark Console →
2a Create the app
Developer Console → Create Custom App (创建企业自建应用)

Paste these values into the form:

GitHub Sync Bot
Syncs GitHub org members, repos, and event notifications with Lark

Click Create (确认创建). You'll land on the app dashboard.

2b Copy your App ID and App Secret
Left sidebar → Credentials & Basic Info (凭证与基础信息)

You'll see App ID (App ID) and App Secret (App Secret) at the top. Click the copy icon next to each. Save them — you'll need them in Step 3.

2c Enable the Bot capability
Left sidebar → Add Capabilities (添加应用能力) → Bot (机器人)

Click + Add capability on the Bot card. This lets the app send messages to group chats.

2d Add permissions
Left sidebar → Permissions & Scopes (权限管理)

Use the search box at the top to find and activate each permission:

Search forPermission namePurpose
contact:user.baseGet basic user info (获取用户基本信息)Look up users by email
contact:departmentGet department basic info (获取部门基本信息)Read departments
contact:contactUpdate contact info (更新通讯录)Manage department membership
im:chatManage group chats (管理群组)Create group chats
im:messageSend messages as bot (以应用身份发消息)Send notification cards

After adding all 5, click Batch Activate (批量开通) at the top of the list. If some permissions require admin approval, a request will be sent to your Lark admin.

Admin? If you are the Lark admin (likely as CEO), the permissions are approved instantly. Otherwise, ask your admin to approve them in the Lark Admin Console (管理后台) → App Permissions.
2e Create a version & set availability
Left sidebar → Version Management & Release (版本管理与发布)

1. Click Create a version (创建版本)

2. Set Availability (可用范围) to All employees (全部员工)

3. Click Save then Submit for release (申请发布)

Admin? If you are the Lark admin, go to Lark Admin Console (管理后台) → Workplace (工作台) → App Review (应用审核) and approve the release.
3
Set 3 secrets on your repo ~1 min

Go to your repo's Settings → Secrets and variables → Actions → New repository secret and add:

Secret nameValue
LARK_APP_IDFrom the Lark app you just created
LARK_APP_SECRETFrom the Lark app you just created
SYNC_GITHUB_TOKENA GitHub PAT — click below to create one with pre-filled scopes
Open repo Secrets → Create GitHub Token →
Tip: The "Create GitHub Token" link pre-fills the name and scopes. Just click Generate token and paste it as SYNC_GITHUB_TOKEN.
4
Run Initial Setup ~1 min

Go to Actions → 🚀 Initial Setup → Run workflow. Fill in:

InputWhat to enter
github_orgYour GitHub organization name
lark_domainfeishu (China) or lark (international)
lark_department_id0 for all employees, or a specific department ID
Open Actions tab →

The workflow will configure everything, sync members, create group chats, and deploy notifications.

✓ You're done!

Everything syncs automatically every day at 02:00 UTC.
New Lark department members get GitHub access automatically.
Every repo push, PR, and issue sends a Lark notification.
